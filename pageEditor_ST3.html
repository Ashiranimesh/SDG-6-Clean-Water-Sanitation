Feedback Page

Technical Description

The Feedback page (feedback.html) implements a fully structured HTML5 form using semantic grouping elements and client-side JavaScript validation. The page is designed to collect user responses about BLUEDROP's environmental programmes.

  • Form Structure: Two <fieldset> elements logically group inputs — one for Personal Details (name, email, phone, address) and one for Feedback Details (programme selection, visit date, submission time, and a message box).
  • Input Types Used: type="text", type="email", type="tel", type="date", type="datetime-local", <textarea>, and <select> to cover a variety of HTML5 input controls.
  • Character Counter: A live character counter updates remaining characters as the user types in the message field: 200 - textarea.value.length. This is implemented via a JavaScript input event listener.
  • JavaScript Validation: On form submission, the script checks that name, email, and message are not empty, and performs a basic email format check using .includes("@") and .includes("."). e.preventDefault() halts submission if validation fails.

Accessibility

Every input has an explicit <label> linked via the for attribute matching the input's id, ensuring screen reader compatibility. The <fieldset> and <legend> elements provide logical context grouping for assistive technologies. The character counter is placed in a visible <p> element adjacent to the textarea so it is discoverable by all users.

Team Page

Technical Description

The Team page (team.html) presents all four project members using a responsive card-based layout. Each card is marked up as an <article> element to reflect its self-contained nature as a piece of content.

  • Flexbox Layout: The .team-container uses display: flex with flex-wrap: wrap and justify-content: center, allowing cards to reflow into a single column on smaller viewports without a media query override.
  • CSS Hover Effect: A transform: scale(1.05) with transition: 0.3s is applied on article:hover, giving each card a subtle lift on interaction.
  • Circular Images: Member photos use border-radius: 50% to display as consistent circular portraits regardless of image dimensions.
  • JavaScript Click Interaction: An event listener is attached to all article elements via querySelectorAll. Clicking any card triggers an alert() displaying the member's name, demonstrating basic DOM traversal with this.querySelector("h3").textContent.

Accessibility

All team member images include descriptive alt text. The student role labels use <h2> elements (.student-label) within each article, maintaining a logical heading hierarchy. The global shared header provides keyboard-navigable links to all site sections.

Content Page

Technical Description

The individual content page (content_ST3.html) uses semantic HTML5 landmarks and internal anchor links for in-page navigation. It features a "Go to Top" button with position: fixed for persistent accessibility across long-scrolling content sections.

Challenges & Lessons Learned

One significant challenge was ensuring the form validation covered all edge cases without relying on HTML5 required attributes alone. I learned to use e.preventDefault() correctly to stop default form submission behaviour when errors are detected.

On the Team page, achieving consistent card heights across team members with different amounts of descriptive text required experimenting with flexbox alignment properties. I resolved this by applying align-items: stretch implicitly through the flex container defaults.

Integrating the shared style.css template with page-specific overrides taught me the importance of CSS specificity — locally scoped styles in <style> blocks took precedence cleanly without breaking the global layout.
