💭 Reflection on Validation

Validating my pages using the W3C Markup Validation Service was an instructive process. The most common issues I encountered during development and subsequently corrected included:

  • Missing lang attribute on the <html> element — corrected by adding lang="en" to all pages.
  • Duplicate id attributes in an early AIS draft — resolved by ensuring each id was unique across the document.
  • An improperly nested <div> inside a <ul> — corrected by restructuring using <article> elements.

Through validation I learned that writing valid HTML directly improves browser compatibility, screen-reader accessibility, and long-term maintainability. The process reinforced good habits: always closing tags, providing alt text for images, and using semantic elements appropriately.
