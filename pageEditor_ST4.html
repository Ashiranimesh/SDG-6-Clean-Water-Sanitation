User Profile Page

Technical Description

The User Profile page (profile.html) demonstrates the use of JavaScript-driven interaction to progressively build and display a user profile. Instead of a traditional form, this page guides users through a series of JavaScript prompts.

  • DOM Manipulation: Used to reveal profile sections dynamically as steps are completed.
  • Progress Indicator: A visual completion bar updates based on the steps finished using a percentage calculation: Math.round((profDone.size/3)*100).

Accessibility

Accessibility is supported through aria-live="polite" for status updates so screen readers announce completion changes. I also ensured the interface is readable and provides clear descriptive feedback messages.

Sitemap Page

Technical Description

The Sitemap (sitemap.html) is implemented using Scalable Vector Graphics (SVG) to map the website hierarchy. This ensures the diagram is responsive and retains clarity at any zoom level.

  • SVG Architecture: Nodes (rectangles and ellipses) represent pages, connected by lines to show relationships.
  • Interactivity: CSS transitions handle hover effects, providing visual feedback on interactive nodes.

Accessibility

The SVG is keyboard-navigable via Tab navigation, and each node includes a <title> for screen reader access.

Content Page

Technical Description

The individual content page (content_ST4.html) uses semantic HTML5 landmarks and internal anchor links for navigation. It features a "Go to Top" button implemented with position: fixed for persistent access.

Validation Report | View Live Page

Challenges & Lessons Learned

A primary challenge was ensuring SVG nodes were keyboard-focusable, resolved by adding tabindex="0" and focus styles. I also learned the security value of using textContent to prevent XSS risks during DOM updates.

Compliance

All pages comply with JANET regulations by strictly hosting academic content related to SDG 6. All external assets are royalty-free and properly cited to maintain academic integrity.
